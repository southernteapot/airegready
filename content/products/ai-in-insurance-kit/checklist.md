## Phase 1 — Scope

- [ ] **1.1** AI System Inventory covers all material insurance AI use cases (underwriting, rating, marketing, claims, fraud, customer service, HR, investments)
- [ ] **1.2** Each system tagged for applicable regulators (NAIC bulletin states, CO DOI, NY DFS, CA DOI, TX DIR, etc.)
- [ ] **1.3** Each system mapped to applicable rules (SB 21-169, Circular 7, NAIC Model Bulletin, EU AI Act, CAIA, HIPAA)
- [ ] **1.4** Provider / deployer / agent role documented

## Phase 2 — Written AI Program (per NAIC Model Bulletin)

- [ ] **2.1** AI Program document adopted and signed by senior management
- [ ] **2.2** Scope, objectives, and roles clearly defined
- [ ] **2.3** Program reviewed by the board or designated committee
- [ ] **2.4** Program reviewed at least annually

## Phase 3 — Governance and roles

- [ ] **3.1** Senior management AI committee chartered and meeting
- [ ] **3.2** Three lines of defense in place (business / risk / audit)
- [ ] **3.3** Named AI program owner with budget and authority
- [ ] **3.4** Reporting to the board on AI posture

## Phase 4 — Documentation

- [ ] **4.1** Each material AI system has a model/system card with purpose, data, methodology, assumptions, limitations, performance, testing, monitoring
- [ ] **4.2** Documentation supports reconstruction of any AI-assisted decision
- [ ] **4.3** Documentation retained per records policy

## Phase 5 — Testing

- [ ] **5.1** Accuracy / performance testing performed at development
- [ ] **5.2** Robustness / stress testing performed
- [ ] **5.3** Bias / unfair discrimination testing across protected classes
- [ ] **5.4** Proxy analysis for non-protected features correlated with protected classes
- [ ] **5.5** Comparison to challenger models where applicable
- [ ] **5.6** Testing repeated at defined cadence
- [ ] **5.7** Results documented and reviewed; mitigations tracked

## Phase 6 — Monitoring

- [ ] **6.1** Performance monitoring in production
- [ ] **6.2** Drift detection and re-training triggers
- [ ] **6.3** Outcome monitoring for disparate impact
- [ ] **6.4** Complaint tracking tied to models
- [ ] **6.5** Threshold-based escalation to governance

## Phase 7 — Third-party AI

- [ ] **7.1** Vendor risk assessment for every external AI
- [ ] **7.2** Vendor contracts address: model documentation, testing, change notice, cooperation in exams, indemnification
- [ ] **7.3** Vendor testing reviewed for independence
- [ ] **7.4** Ongoing vendor oversight and re-assessment

## Phase 8 — Consumer-facing obligations

- [ ] **8.1** Notice where a high-risk AI system is a substantial factor in a consequential decision (CAIA scope)
- [ ] **8.2** Adverse-action / denial notices with specific reasons
- [ ] **8.3** Appeal path to human review
- [ ] **8.4** Data correction mechanism
- [ ] **8.5** Chatbot / bot disclosure compliant with EU AI Act Art 50, TX TRAIGA, CA SB 1001

## Phase 9 — Rate filings

- [ ] **9.1** Rate filings disclose AI use where required by state
- [ ] **9.2** Actuarial documentation supports model outputs
- [ ] **9.3** Filings updated when material model changes occur
- [ ] **9.4** Responses prepared for DOI questions on AI

## Phase 10 — Claims AI

- [ ] **10.1** Automated adjudication paths documented with human-review triggers
- [ ] **10.2** Fraud AI has remedy path for false positives
- [ ] **10.3** Denial explanations specific and actionable
- [ ] **10.4** Claims-handling timeliness monitored under unfair claims practices laws
- [ ] **10.5** Customer communication disclosed as AI where applicable

## Phase 11 — State-specific compliance

### Colorado SB 21-169 (life insurance)

- [ ] **11.1** Risk management framework document on file
- [ ] **11.2** Reporting to CO DOI per schedule
- [ ] **11.3** Testing for unfair discrimination across protected classes
- [ ] **11.4** External consumer data use documented

### New York DFS

- [ ] **11.5** Circular Letter 7 obligations mapped
- [ ] **11.6** Part 500 cybersecurity covers AI systems

### California DOI

- [ ] **11.7** Notice Issued about AI use where required
- [ ] **11.8** Alignment with CA CCPA/CPRA ADMT rules

### Other states

- [ ] **11.9** State-by-state review completed and tracked

## Phase 12 — EU AI Act (if EU customers)

- [ ] **12.1** Life and health insurance risk assessment classified Annex III high-risk
- [ ] **12.2** Technical documentation, QMS, data governance, logs, transparency, human oversight, accuracy/robustness/cybersecurity
- [ ] **12.3** Deployer FRIA per Art 27
- [ ] **12.4** CE marking, EU database, serious incident reporting (Art 73)

## Phase 13 — Records and audit readiness

- [ ] **13.1** Model documentation, validation reports, testing, monitoring, complaints, incidents retained
- [ ] **13.2** Market-conduct-exam binder ready (AI Program, inventory, testing samples, third-party oversight, complaint log)
- [ ] **13.3** Retention period aligned with insurance records rules (typically 5–10+ years)
