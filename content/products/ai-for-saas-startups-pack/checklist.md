# AI for SaaS Startups — Focused Readiness Checklist

**Company:** ______________________________
**Stage:** ☐ Pre-seed  ☐ Seed  ☐ Series A  ☐ Series B+
**Completed by:** ______________________________
**Date:** ______________________________

> Scaled to startup reality. If you're selling to enterprises, aim to hit everything in Phases 1-4 before first enterprise deal and everything in Phases 1-7 before Series A close.

---

## Phase 1 — Founding-stage minimum

- [ ] **1.1** AI vendor selection uses enterprise-tier terms (no training on inputs)
- [ ] **1.2** DPAs signed with AI vendors
- [ ] **1.3** Data flow diagram drafted
- [ ] **1.4** Privacy policy mentions AI use
- [ ] **1.5** Customer-facing disclosure for AI features
- [ ] **1.6** Founders understand the top 3 AI risks for their product

## Phase 2 — Before first customer

- [ ] **2.1** Internal AI Acceptable Use Policy (short form)
- [ ] **2.2** AI System Inventory (spreadsheet fine)
- [ ] **2.3** Approved tools list
- [ ] **2.4** Basic incident response plan with AI scenarios
- [ ] **2.5** Security questionnaire responses prepared (top 20 items)

## Phase 3 — Before first enterprise customer

- [ ] **3.1** Customer MSA / DPA template covers AI
- [ ] **3.2** Sub-processor list published
- [ ] **3.3** No-training-on-customer-data contractual commitment
- [ ] **3.4** Model version / change-notice commitment
- [ ] **3.5** Kill switch for AI features exists and is tested
- [ ] **3.6** Product-side disclosures for AI features

## Phase 4 — Series A readiness

- [ ] **4.1** AI governance doc (can be lightweight)
- [ ] **4.2** Impact assessment template for high-stakes use cases
- [ ] **4.3** Red-team or adversarial testing documented
- [ ] **4.4** Subprocessor and vendor oversight process
- [ ] **4.5** Monitoring and usage alerts in place
- [ ] **4.6** Training completed for employees
- [ ] **4.7** Investor diligence binder available (AI section)

## Phase 5 — Enterprise buyer + SOC 2

- [ ] **5.1** SOC 2 Type I or II in progress / completed
- [ ] **5.2** AI controls mapped to SOC 2 CC categories
- [ ] **5.3** Trust Center / Security page on website
- [ ] **5.4** Security questionnaire automation (e.g., Vanta, Drata, SafeBase)

## Phase 6 — State and EU compliance

- [ ] **6.1** Colorado AI Act scope analysis (consequential-decision categories)
- [ ] **6.2** California — SB 942 / AB 2013 / CCPA ADMT applicability
- [ ] **6.3** Texas TRAIGA applicability
- [ ] **6.4** NYC LL 144 for hiring
- [ ] **6.5** EU AI Act applicability (user base / hosting)
- [ ] **6.6** GDPR / UK GDPR posture

## Phase 7 — Product guardrails

- [ ] **7.1** Rate limiting on AI endpoints
- [ ] **7.2** Abuse detection and monitoring
- [ ] **7.3** Input validation for prompt injection risk
- [ ] **7.4** Output filters for prohibited content
- [ ] **7.5** PII redaction where appropriate
- [ ] **7.6** Logging that doesn't itself create a liability (avoid storing raw sensitive prompts long-term)

## Phase 8 — People

- [ ] **8.1** AI-aware hiring process (no accidental NYC LL 144 violations)
- [ ] **8.2** AI AUP training for new hires
- [ ] **8.3** Access controls (SSO, MFA) on AI tools

## Phase 9 — Incident posture

- [ ] **9.1** Playbook for prompt-injection data leaks
- [ ] **9.2** Playbook for cross-tenant output leaks
- [ ] **9.3** Playbook for vendor outage
- [ ] **9.4** Playbook for FTC or state AG inquiry
- [ ] **9.5** PR-ready statement template

## Phase 10 — Board / investor reporting

- [ ] **10.1** Quarterly or semi-annual AI update on board deck
- [ ] **10.2** Material AI incidents surfaced to board
- [ ] **10.3** Regulatory watch item in board materials
