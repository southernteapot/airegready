# Incident / Complaint Mini-Log

Purpose: Track user complaints, bad outputs, privacy concerns, data mistakes, or AI-related issues so they do not disappear into inboxes, chat threads, support tools, or memory.

## When To Use It

Use this whenever someone reports an incorrect, confusing, harmful, biased, unexpected, privacy-related, or AI-related issue. Also use it for near misses, internal discoveries, client complaints, platform notices, support tickets, and accidental sensitive-data use.

## What This Produces

You should finish with a lightweight issue record showing what happened, what data was involved, what you did, whether anything changed, and whether follow-up is needed.

Add a row as soon as you learn about the issue. You can update the row later as you investigate, respond, fix, or close it.

## Issue Types

Use these labels or create your own:

- Incorrect output
- Harmful or unsafe output
- Biased or offensive output
- User confusion
- Privacy or data concern
- Accidental sensitive-data use
- Client confidential data concern
- Disclosure concern
- Unsupported claim
- Vendor, outage, or rate-limit issue
- Prompt or model behavior issue
- Security or access issue
- Platform or marketplace notice
- Other

## Response Notes

For each meaningful issue, capture:

- What happened?
- Who was affected?
- What data was involved?
- Was the user, client, customer, platform, or vendor contacted?
- Was any public copy, disclosure, FAQ, prompt, workflow, or vendor setting updated?
- Does the issue suggest a broader pattern?
- Should the project be paused, narrowed, tested, or reviewed by someone qualified?

## Sensitive-Data Note

If sensitive data is accidentally pasted into an AI tool:

1. Stop using that chat, prompt, file, or workflow.
2. Save the tool name, date, data type, and what happened.
3. Check whether the tool allows deletion or retention controls.
4. Check whether the data was covered by a client contract, privacy promise, NDA, platform rule, or law.
5. Ask qualified help if the data was sensitive, regulated, confidential, client-owned, credential-related, payment-related, or child-related.

## Escalate Or Get Help If

- The issue involves sensitive personal data, client confidential information, credentials, payment data, children, or regulated data.
- The AI output may have affected a real-world decision.
- A user claims harm, financial loss, discrimination, privacy harm, or reliance on incorrect output.
- The issue involves legal, medical, mental health, financial, employment, housing, credit, education, insurance, safety, identity, or access topics.
- The same issue happens more than once.
- You are unsure whether to notify someone, preserve records, pause a feature, or change data handling.
