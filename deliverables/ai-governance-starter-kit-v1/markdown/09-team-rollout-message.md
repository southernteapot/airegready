Hi team,

We are putting a lightweight AI governance file in place so our AI use is easier to manage and explain.

- Which AI tools are approved for work use
- What data can and cannot go into those tools
- When a human needs to review AI output
- How to request a new AI tool or workflow
- Where we track changes over time

Please review these documents by [DATE]:

- AI Acceptable Use Policy: [LINK]
- Green / Yellow / Red AI Use Rules: [LINK]
- Approved AI Tools List / Inventory: [LINK]
- New AI Tool Request Path: [LINK]

The short version:

- Use approved tools for approved purposes.
- Do not paste confidential, customer, employee, student, health, financial, legal, regulated, credential, or security-sensitive data into AI tools unless that use is approved.

## Short FAQ to include or link

### Are we banning AI?

No. We are making AI use visible and setting basic rules so useful AI work can continue responsibly.

### Can I keep using tools I already use?

Maybe. Add them to the inventory or approved tools list. If the tool is not approved yet, do not use it with sensitive or confidential data until reviewed.

### What counts as sensitive data?

Customer data, employee data, student data, patient or health data, financial data, legal data, biometric data, credentials, source code or secrets, confidential business information, and anything under an NDA or contract restriction.

### What if I already pasted restricted data into a tool?

Report it to [CONTACT] so we can understand what happened and decide the next step. Do not repeat the upload while it is under review.

### Who approves new AI tools?

[APPROVER / TEAM] reviews new tool requests. Use [REQUEST PATH / FORM LINK].
