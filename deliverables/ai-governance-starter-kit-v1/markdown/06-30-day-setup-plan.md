> **How to use.** This plan turns the starter kit into four weeks of practical work. It is built for small teams that need a credible first pass, not a mature enterprise program.

---

## Before you start

Choose one person to own the first pass. That person does not need to be a lawyer or compliance officer. They need enough authority to collect tool information, ask teams for input, and schedule follow-up decisions.

**Suggested owner:** operations lead, founder, chief of staff, IT lead, privacy/security lead, or department head.

---

## Week 1 - Make AI use visible

**Primary document:** AI Use Inventory

### Goals

- Find the AI tools already in use
- Identify owners and teams
- Mark obvious sensitive-data issues
- Separate approved, under-review, and unknown tools

### Tasks

- [ ] Ask each team to list AI tools they use for work
- [ ] Include personal AI accounts used for work tasks
- [ ] Add every known tool to the AI Use Inventory
- [ ] Record vendor, owner, use case, data inputs, and approval status
- [ ] Flag tools that process customer, employee, student, health, financial, legal, confidential, or regulated data
- [ ] Identify any customer-facing or decision-support use

### End-of-week output

- A draft inventory with known tools and owners
- A short list of unknowns to resolve
- A list of tools or workflows that need deeper review

### Week 1 meeting prompt

Which AI tools surprised us, and which ones need immediate restrictions or review?

---

## Week 2 - Set the basic rules

**Primary document:** AI Acceptable Use Policy

### Goals

- Define approved tools
- Write simple data rules
- Define prohibited uses
- Decide when human review is required
- Identify who approves exceptions

### Tasks

- [ ] Fill in company name, owner, effective date, and policy owner
- [ ] List approved AI tools and permitted uses
- [ ] Adapt the Green / Yellow / Red AI Use Rules
- [ ] Decide what staff may never put into AI tools without approval
- [ ] Define when AI output must be reviewed before use
- [ ] Define escalation contacts for uncertain or higher-risk use
- [ ] Decide whether staff acknowledgment is required

### End-of-week output

- A draft AI acceptable use policy
- A short staff-facing rules summary
- A list of policy questions that need leadership or counsel review

### Week 2 meeting prompt

What rule would prevent the most likely bad AI use in our team this month?

---

## Week 3 - Add review for new or risky use

**Primary documents:** AI Risk Tiering Decision Tree, AI Risk Intake and Impact Assessment, and AI Tool Approval Mini-Checklist

### Goals

- Create a review path for new AI tools and workflows
- Decide which requests can use the mini-checklist
- Decide which requests need deeper review
- Record approval conditions

### Tasks

- [ ] Choose who reviews new AI tool requests
- [ ] Use the risk tiering decision tree for one proposed tool or workflow
- [ ] Use the mini-checklist for one low-risk tool request as a test
- [ ] Use the risk intake and impact assessment for one higher-risk workflow or obvious gap
- [ ] Define stop-and-escalate triggers
- [ ] Add approved conditions back to the inventory
- [ ] Add material risks to the risk register
- [ ] Set next review dates for approved tools

### End-of-week output

- A working request path for new AI use
- One completed sample mini-checklist, tiering record, or intake/impact assessment
- Clear escalation triggers for sensitive or higher-risk use
- A starter risk register for the risks that need follow-up

### Week 3 meeting prompt

What kinds of AI use should we allow quickly, and what kinds should require review first?

---

## Week 4 - Roll out and start the update rhythm

**Primary documents:** Team Rollout Message, First Governance Meeting Agenda, AI Update Tracker, What Good Looks Like

### Goals

- Tell staff what changed
- Start the update log
- Assign recurring owners
- Decide what to improve next

### Tasks

- [ ] Send the Team Rollout Message
- [ ] Publish the approved tools list and rules where staff can find them
- [ ] Hold the first governance meeting
- [ ] Start the AI Update Tracker
- [ ] Record open questions and next actions
- [ ] Schedule the first 30-day follow-up
- [ ] Use What Good Looks Like to check whether the first pass is complete

### End-of-week output

- Staff communication sent
- Update tracker started
- Open actions assigned
- Next review meeting scheduled

### Week 4 meeting prompt

What is the smallest next improvement that would reduce the most AI risk or confusion?

## If you get stuck

- If you cannot identify all AI tools, start with the obvious ones and keep a running open item.
- If legal questions come up, mark them for qualified review instead of guessing.
- If staff are using tools that are not approved, document the use first, then decide whether to approve, restrict, or replace it.
- If the policy feels too long, publish the one-page rules first and improve the longer policy later.
