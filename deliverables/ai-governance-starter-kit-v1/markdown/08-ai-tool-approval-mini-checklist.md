## 2. Stop triggers

If any answer below is **yes**, pause. Use the AI Risk Tiering Decision Tree and route the request to the AI Risk Intake and Impact Assessment or a qualified reviewer.

| Question | Yes / No | Notes |
| --- | --- | --- |
| Will the tool process customer, employee, student, patient, financial, health, biometric, legal, or regulated data? |  |  |
| Will it materially influence decisions about people? |  |  |
| Will output be shown directly to customers, applicants, patients, students, or the public? |  |  |
| Will it connect to email, CRM, files, code repositories, databases, production systems, or other internal systems? |  |  |
| Will it act autonomously, send messages, make purchases, change records, or trigger workflows? |  |  |
| Will it be used in hiring, lending, housing, insurance, healthcare, education, legal, government, or safety contexts? |  |  |
| Does the vendor use customer inputs to train models by default? |  |  |
| Is the vendor missing basic privacy, terms, or security information? |  |  |

**If any stop trigger is yes:** do not approve through this mini-checklist. Use the AI Risk Tiering Decision Tree and the AI Risk Intake and Impact Assessment.
