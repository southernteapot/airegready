**Requesting team / owner:** [TEAM / OWNER]  
**Date:** [DATE]  
**Completed by:** [NAME]

Use this checklist before approving a new AI tool for company work. It is meant for quick triage, not a full vendor review.

---

## 1. Basic purpose

- [ ] What does the tool do?

> [SHORT DESCRIPTION]

- [ ] What problem does it solve?

> [PROBLEM / USE CASE]

- [ ] Who will use it?

> [TEAM / ROLES / NUMBER OF USERS]

- [ ] Is there already an approved tool that does this?

> [YES / NO / EXPLAIN]

---

## 2. Data check

What data will users enter? Check all that apply:

- [ ] Public information only
- [ ] Internal business information
- [ ] Customer personal data
- [ ] Employee personal data
- [ ] Health, financial, biometric, children’s, or government ID data
- [ ] Source code, contracts, pricing, trade secrets, or confidential business information
- [ ] Third-party data received under NDA or contract

**Stop and escalate** if anything sensitive, confidential, regulated, or personal will be entered.

---

## 3. Use check

- [ ] Will output be shown directly to customers, clients, patients, students, applicants, employees, or the public?
  - [ ] No
  - [ ] Yes — describe: [DESCRIPTION]

- [ ] Will the tool make or materially influence decisions about people?
  - [ ] No
  - [ ] Yes — describe: [DESCRIPTION]

- [ ] Will the tool connect to company systems, files, email, CRM, HR, support, finance, production code, or databases?
  - [ ] No
  - [ ] Yes — describe: [DESCRIPTION]

Escalate before approval if any answer is yes.

---

## 4. Vendor check

- [ ] Vendor has a public privacy policy.
- [ ] Vendor has clear terms of service.
- [ ] Vendor explains whether customer inputs are used for model training.
- [ ] Vendor explains data retention and deletion.
- [ ] Vendor uses encryption in transit.
- [ ] Vendor has reasonable security information available, such as SOC 2, ISO 27001, security page, or equivalent.
- [ ] Pricing and contract terms are clear.
- [ ] Exit path is clear if the company stops using the tool.

---

## 5. Decision

Recommended action:

- [ ] Approve for limited use
- [ ] Approve with conditions
- [ ] Needs full vendor/risk review
- [ ] Reject
- [ ] Need more information

Conditions, if any:

> [CONDITIONS]

Approved by: [NAME]

## 6. If approved

- [ ] Add tool to the AI Tool Inventory.
- [ ] Add tool to the approved tools list.
- [ ] Tell users what data may and may not be entered.
- [ ] Configure SSO/MFA if available.
- [ ] Schedule review date.
